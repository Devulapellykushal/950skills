# GDPR Compliance

## When to use

Use this skill when the user needs **GDPR** implementation or compliance. Triggers: "GDPR", "data protection", "right to erasure", "consent", "DPIA", "breach notification", "privacy by design", "data subject rights".

## Overview

Implement technical and process measures for GDPR: lawful basis, consent, data subject rights (access, erasure, portability, etc.), DPIAs, breach notification, retention, and privacy by design.

## Deliverables

1. **Lawful basis** – Document basis per processing (consent, contract, legal obligation, legitimate interest); record-keeping; consent where required.
2. **Consent** – Clear, specific, opt-in; withdraw as easy as give; record consent (who, when, what); no pre-ticked boxes.
3. **Data subject rights** – Access (Art. 15), rectification (16), erasure (17), restrict processing (18), portability (20), object (21); process within 30 days; identity verification.
4. **DPIAs** – When high risk (profiling, sensitive data, systematic monitoring); describe processing, necessity, risks, mitigations; consult DPA if needed.
5. **Breach notification** – Detect and document; notify DPA within 72h where risk; notify individuals if high risk; log and review.
6. **Retention** – Define retention per purpose; automated deletion or anonymization when no longer needed; document in privacy notice.
7. **Privacy by design** – Minimization; purpose limitation; default settings privacy-friendly; encryption and access controls; document in architecture and DPIA.

## Principles

- One skill to cover GDPR technical and process requirements; link to legal counsel for interpretation.
- Maintain a processing record and data flow; keep privacy notices and consent flows up to date.
- Test data subject request handling and breach procedures regularly.
