# SOC 2 Compliance

## When to use

Use this skill when the user needs **SOC 2** readiness or control implementation. Triggers: "SOC 2", "SOC2", "trust service criteria", "audit readiness", "control mapping", "security availability confidentiality", "TSC".

## Overview

Implement and document controls aligned to AICPA SOC 2 Trust Service Criteria (security, availability, processing integrity, confidentiality, privacy) for audit readiness.

## Deliverables

1. **Trust Service Criteria** – Security (CC6.1, access; CC7.1, monitoring; etc.); Availability (A1); Processing Integrity (PI1); Confidentiality (C1); Privacy (P1–P9) as applicable.
2. **Control mapping** – Map each in-scope criterion to technical and process controls; document evidence (policies, config, logs, screenshots); owner and frequency.
3. **Access control** – Least privilege; MFA; access reviews; role-based access; logging and review of access changes.
4. **Monitoring and incident response** – Logging of security events; alerting; incident response plan and runbooks; post-incident review.
5. **Change management** – Approved changes; testing; rollback; separation of duties where required.
6. **Vendor risk** – Due diligence for subservice organizations; contracts and SOC reports from vendors.
7. **Audit readiness** – Evidence collection; gap remediation; readiness assessment; auditor walkthrough support.

## Principles

- Document controls and evidence in one place; keep it current.
- Align technical controls (e.g. IAM, encryption, logging) to specific criteria.
- Plan for at least one full cycle of evidence before audit.
